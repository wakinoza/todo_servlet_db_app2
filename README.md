# MySQL連携 Todo アプリケーション - Security Update Edition

本プロジェクトは、以前公開した「Servlet/JSP ToDoアプリ」をベースに、実務レベルの堅牢性と環境構築の柔軟性を備えるための大規模セキュリティアップデートを施したリポジトリです。

## 🛡️ アップデートの概要
情報処理推進機構の『安全なウェブサイトの作り方』などを参考に、包括的なセキュリティ強化を行いました。

### 1. 根本的な脆弱性対策
SQLインジェクション対策（徹底とさらなる強化）:
  - プリペアドステートメントの利用（既存）: 初期段階より全てのDB操作において静的プレースホルダの利用を徹底し、安全なSQL実行を担保しています。
  - サーバー側実行の強制（新規）: JDBC接続URLに `useServerPrepStmts=true` を追加し、MySQLサーバー側での静的プレーすホルダを強制。
  - エンコーディングの明示（新規）: `characterEncoding=UTF-8` を指定し、バイパス攻撃リスクを排除しました。

XSS (Cross-Site Scripting) 対策:
  - 全てのJSP出力（hiddenフィールド含む）に `<c:out>` を徹底。
  - レスポンスヘッダに `Content Security Policy (CSP)` を設定し、意図しないスクリプト実行を遮断。
  - `HttpOnly` 属性の付与によるクッキー保護。

### 2. 認証・認可およびセッション管理の強化
パスワードのハッシュ化:
  - `BCrypt` を利用したユーティリティクラスを実装。データベースに平文パスワードを保存しない構成へ変更。

CSRF (Cross-Site Request Forgery) 対策:
  - 認証管理をセッションベースへ移行し、各フォーム送信時にワンタイムトークンを照合する機能を実装。

セッション固定化攻撃対策:
  - ログイン認証成功時に旧セッションを破棄し、新規セッションIDを発行する処理を追加。

スニッフィング対策:
  - `X-Content-Type-Options: nosniff` や `X-Frame-Options` 等のセキュリティヘッダをServlet側で一括指定。

### 3. インフラ・構成管理のモダン化
Dockerによる環境構築のコード化 (IaC):
  - MySQL 8.0 / Tomcat 10 のコンテナ構成へ移行し、開発環境のポータビリティを向上。

機密情報の完全分離:
  - `JNDI` と環境変数を組み合わせ、DBパスワード等の機密情報をソースコード（Git管理下）から完全に排除しました。


## ディレクトリ構造
```
.
├── src/main/java    # Javaソースコード
│   ├── bean/        # エンティティ（データモデル）
│   ├── dao/         # DBアクセス (1人作業用アカウントへ統一済み)
│   ├── model/       # ビジネスロジック
│   ├── servlet/     # コントローラ (セキュリティヘッダ・セッション管理実装)
│   └── util/        # 共通ユーティリティ (BCryptによるハッシュ化)
├── src/main/webapp  # Webリソース
│   ├── WEB-INF/
│   │   └── jsp/     # 各種JSP画面 (c:outによるXSS対策を徹底)
│   └── index.jsp    # 初期表示画面
├── compose.yml      # Dockerコンテナ構成定義 (.envから環境変数を注入)
├── Dockerfile       # Tomcatイメージのビルド定義
├── context.xml      # Tomcat環境設定 (JNDIリソース定義・環境変数対応)
├── init.sql         # DB初期化 (テーブル作成・初期ユーザー投入)
├── pom.xml          # Maven依存関係管理
├── .env.example     # 環境変数テンプレート
└── .gitignore       # Git管理除外設定 (.env, target, IDE設定ファイル)
```

## 🚀 起動手順
Dockerがインストールされた環境であれば、以下の手順で動作確認が可能です。

1. リポジトリをクローン
```bash
git clone https://github.com/wakinoza/todo_servlet_db_app2
cd todo_servlet_db_app2
```

2. 環境変数の準備 .env.example をコピーして .env を作成し、必要な値を設定してください。
```bash
cp .env.example .env
```

3. コンテナの起動
```bash
docker compose up -d --build
```

4. アクセス ブラウザで http://localhost:8080/todo/Main にアクセスしてください。
以下の情報を入力すると、ログイン可能です。

- ユーザー名：yamada
- パスワード：yamada_password

## 苦戦した点と設計上の判断

初期化スクリプトにおける環境変数の扱い：
当初、init.sql 内で環境変数を用いて複数の作業用ユーザーを動的に作成することを検討しました。しかし、コンテナ起動時のシェルスクリプトによる置換（envsubst等）を導入すると、構成が複雑化し、かえって機密情報の管理漏れや保守性の低下を招くリスクがあると判断しました。

判断: 今回は「セキュリティの確実性とシンプルさ」を最優先し、Docker公式の環境変数機能を最大限活用したシングル作業ユーザー構成を採用しました。これにより、Git管理下のファイルに機密情報を1文字も残さない、実務的なクリーンな構成を実現しています。
