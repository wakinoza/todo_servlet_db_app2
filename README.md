# MySQL連携 Todo アプリケーション

## アプリケーション概要

本アプリケーションは、Java Servlet/JSP を用いて開発されたTODO管理Webアプリケーションです。
従来のメモリ管理型TODOアプリに MySQL を組み込み、データの永続化を実現しました。

ブラウザを閉じてもTODO情報が保持されるようになり、実用性を意識したWebアプリケーションとして設計されています。

## 主な機能
1. ログイン認証
ユーザー名とパスワードによる簡易認証機能を実装しています。

2.ユーザー名の表示
ログイン後の画面に、現在ログインしているユーザー名を表示します。

3.TODOの新規作成
入力フォームからTODOを登録し、MySQLデータベースに保存します。

4.TODOの一覧表示
MySQLに保存されたTODOを取得し、一覧形式で表示します。

5.進捗管理機能
TODOの状態を「未実施」「実施中」「完了済」で管理
ボタン操作で進捗状態を更新可能

6.TODOの削除
完了済のTODOを削除可能
データベースから該当データを削除

## 開発で工夫・苦労した点

MySQLをEclipseから利用する際、JDBCドライバが認識されず、
EclipseがMySQLを認識しない問題に直面しました。

原因は「ビルドパスが正しく設定されていなかったこと」でした。

この問題を解決するために、

1. Javaのクラスパスとビルドパスの仕組みを再学習

2. JDBCドライバ（Connector/J）を適切に追加

3. Eclipse上で外部JARを正しく参照できるように設定

といった手順を踏み、最終的にMySQLとの接続を実現しました。

この経験を通して、「コードだけでなく開発環境を整える力」の重要性を強く実感しました。

## 開発を通じて得られた学び

1. JDBC を用いたRDB操作の理解
データの追加・取得・更新・削除をJavaから直接操作する経験を通じて、
JDBCの基本的な流れを実践的に理解しました。

2.ビルドパスとクラスパスの重要性
これまで抽象的だった「ビルドパス」の仕組みを、
実際のエラー解決を通じて深く理解できました。

3.WebアプリケーションとDBの連携設計
Web層（Servlet）と永続層（MySQL）の責務を意識した設計を意識しました。

4. DB接続情報の一元管理と保守性の向上
データベースの接続情報（URL、ユーザー名、パスワード）を `context.xml` に集約し、アプリケーションコードから分離しました。
これにより、環境ごとの設定変更をコード修正なしで行えるようになり、保守性と再利用性を意識した設計の重要性を学びました。
